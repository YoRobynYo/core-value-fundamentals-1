Module 5: Loops Part 4 — Accumulators 🏗️

🏗️ What is an Accumulator? 🏗️

Imagine you have a savings jar at home. Every day you add a few coins to it. After a week, you open the jar and count up the total. The jar has been accumulating money over time — collecting a little bit each day!

In coding, an accumulator works the same way. It is a variable that sits OUTSIDE a loop and keeps a running total. Every time the loop runs, it adds a little more to the total — and the accumulator remembers it all!

Why Do We Need Accumulators?

Sometimes you need to track a total that builds up over time. For example:

  • How many boxes of corn have arrived this week?
  • How many batches of popcorn have we made today?
  • How much money do we owe our workers?
  • How many defects has the inspector found during their shift?

In every case, you need to start at zero, then ADD to the total each time something happens. The accumulator variable does exactly that!

💡 Key idea: An accumulator is a variable created OUTSIDE the loop. The loop updates it every time it runs. This is different from variables that are created inside the loop — those get reset every time!

📐 The Accumulator Pattern 📐

Let's look at exactly how the accumulator pattern works. There are three parts:

Part 1: Create the accumulator OUTSIDE the loop

First, you create your accumulator variable and set it to zero. This goes BEFORE the loop starts:

SET "total saved" = 0

Part 2: Update the accumulator INSIDE the loop

Inside the loop, you add to the accumulator every time the loop runs:

SET "total saved" = "total saved" + "coins today"

Part 3: Display the accumulator after each update

After adding to the total, you print the result so you can see the running total growing:

PRINT "Total saved:", "total saved"

Putting It All Together:

SET "total saved" = 0
FOR counter FROM 1 TO 7 DO
  SET "total saved" = "total saved" + 5
  PRINT "Day", counter, "Total saved:", "total saved"
END FOR

Output:
Day 1 Total saved: 5
Day 2 Total saved: 10
Day 3 Total saved: 15
Day 4 Total saved: 20
Day 5 Total saved: 25
Day 6 Total saved: 30
Day 7 Total saved: 35

💡 Notice: The variable "total saved" is created ONCE outside the loop, then updated over and over inside the loop. That is the accumulator pattern in action!

🏭 Accumulators at the Factory 🏭

At the Popcorn Factory, accumulators are used every single day. The factory manager needs to track totals across many areas of the operation:

  • Supplies: How many boxes of corn have arrived from all deliveries?
  • Production: How many batches of popcorn have we made in total?
  • Quality: How many defects has the inspector found today?
  • Payroll: What is the total amount we need to pay our workers?
  • Usage: How much salt or oil have we used up this week?

In each case, the pattern is the same:

  1. Create an accumulator variable and set it to zero
  2. Loop through each item (delivery, worker, day, batch, etc.)
  3. Add to the accumulator inside the loop
  4. Print the running total after each update

The accumulator can go UP (adding supplies, production, wages) or DOWN (using up materials from a starting supply). The pattern works the same way either way!

💡 Tip: The variable name should describe what you are tracking — "total wages" tells you exactly what is being accumulated, much better than just calling it "total"!

💡 Why Accumulators Matter 💡

Accumulators might seem simple — just adding numbers in a loop. But they are one of the most powerful patterns in all of coding. Here is why:

Real-World Examples:

  • A shop uses an accumulator to calculate your total bill when you buy several items
  • A fitness tracker uses an accumulator to count your total steps during the day
  • A game uses an accumulator to track your score as you collect points
  • A weather station uses an accumulator to calculate the average temperature over a week

The Power of the Pattern:

Without accumulators, you would have to write a separate line of code for every single item you wanted to total. Ten items? Ten lines. A hundred items? A hundred lines!

With an accumulator and a loop, it does not matter if you are tracking 10 items or 10,000 items — the code stays the same! The loop handles the repetition and the accumulator keeps track of the total. That is the real power of this pattern.

The supply room has several shelves filled with corn boxes. We need to count how many boxes are on each shelf and keep a running total of everything we find. Let's use an accumulator to track the total!

Part A: Counting Corn From the Shelves

There are 4 shelves in the supply room, and each shelf holds 5 boxes of corn. We need to go through each shelf, add the boxes to our running total, and print the updated total each time.

  • Step 1:  Create a variable named "total corn boxes" and set its value to 0.
  • Step 2:  Create a variable named "number of shelves" and set its value to 4.
  • Step 3:  Create a variable named "boxes per shelf" and set its value to 5.
  • Step 4:  Now write the code to track the total:
    SET "total corn boxes" = 0
    SET "number of shelves" = 4
    SET "boxes per shelf" = 5

      SET "total corn boxes" = "total corn boxes" + "boxes per shelf"

    END FOR
