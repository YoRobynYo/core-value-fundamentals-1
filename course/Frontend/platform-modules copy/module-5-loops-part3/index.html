Module 5: Loops Part 3 🔁

🔁 Repeating Tasks in the Factory 🔁

Welcome to Part 3 of Loops! In the factory, many tasks happen over and over again. We count supplies going down. We share items out between machines. We track what's left after each step. Instead of writing the same code again and again, we use a loop to do it all in one go!

Example: Counting Down Oil Containers

The factory starts with 30 oil containers. Each shift, workers use 1 container. We want to see how many are left after each shift. Without a loop, we would have to write it out manually:

SET "remaining" = 30
SET "remaining" = "remaining" - 1
PRINT "Shift 1:", "remaining", "containers remain"
SET "remaining" = "remaining" - 1
PRINT "Shift 2:", "remaining", "containers remain"
SET "remaining" = "remaining" - 1
PRINT "Shift 3:", "remaining", "containers remain"
...and so on for every shift!

That's a lot of copying! Now look what happens with a loop:

SET "remaining" = 30
FOR shift FROM 1 TO 6 DO
  SET "remaining" = "remaining" - 1
  PRINT "Shift", shift, ":", "remaining", "containers remain"
END FOR

One loop replaces all that copying! The loop acts like a factory supervisor, telling the computer: "Do these steps for each shift." Change the number of shifts and the loop adjusts automatically.

🔄 The FOR Loop Pattern 🔄

Let's review the FOR loop structure we use. Every FOR loop has the same basic shape:

FOR variable FROM start TO end DO
  (steps to repeat go here)
END FOR

Breaking it down:

  • FOR — starts the loop
  • variable — a name we choose (like i, shift, or day) that counts which cycle we are on
  • FROM start TO end — the range to loop through (like FROM 1 TO 6)
  • DO — begins the steps that repeat
  • END FOR — marks the end of the repeating steps

Counting with Different Variable Names

The variable inside the loop can be called anything that makes sense:

FOR shift FROM 1 TO 6 DO
  PRINT "Processing shift", shift
END FOR

Output:
Processing shift 1
Processing shift 2
Processing shift 3
Processing shift 4
Processing shift 5
Processing shift 6

Whether you call it i, shift, day, or machine — the loop works exactly the same way. Choose a name that describes what you are counting!

📉 Counting Down with Subtraction 📉

Sometimes in the factory, supplies go DOWN instead of up. Oil containers get used, bags get shipped out, stock gets consumed. We can use a loop with subtraction to track how much is left after each step.

How It Works

  • Step 1: Start with your total amount
  • Step 2: In each loop cycle, subtract the amount used
  • Step 3: Print what remains after each cycle

Example: Tracking Oil Containers

We start with 30 oil containers. Each shift, 1 container is used. Let's track what's left over 6 shifts:

SET "total containers" = 30
FOR shift FROM 1 TO 6 DO
  SET "total containers" = "total containers" - 1
  PRINT "Shift", shift, ":", "total containers", "containers remain"
END FOR

Output:
Shift 1: 29 containers remain
Shift 2: 28 containers remain
Shift 3: 27 containers remain
Shift 4: 26 containers remain
Shift 5: 25 containers remain
Shift 6: 24 containers remain

Each time through the loop, we subtract 1 from the total and print what's left. The value keeps going down — that's what makes it a countdown loop!

💡 Tip: You can subtract any amount — not just 1! If the factory uses 2 containers per shift, just subtract 2 instead of 1 each time through the loop.

➗ Sharing with Division ➗

Another factory task is sharing supplies equally. How many packets does each machine get? How many bags fit on each shelf? Division helps us split things fairly, and a loop lets us try different numbers quickly!

How It Works

  • Step 1: Start with your total amount
  • Step 2: In each loop cycle, divide by the number of recipients
  • Step 3: Print the result for each number

Example: Distributing Flavor Packets

We have 24 flavor packets to share. Let's see how many each machine gets if we have different numbers of machines:

SET "total packets" = 24
FOR machines FROM 1 TO 4 DO
  SET "each gets" = "total packets" / machines
  PRINT machines, "machines: Each gets", "each gets", "packets"
END FOR

Output:
1 machines: Each gets 24 packets
2 machines: Each gets 12 packets
3 machines: Each gets 8 packets
4 machines: Each gets 6 packets

With 1 machine, it gets all 24. With 2 machines, they each get 12. With 4 machines, they each get 6. The loop tries each number and shows us the answer instantly!

💡 Tip: Sometimes division doesn't split evenly. For example, 24 ÷ 5 = 4.8. That's perfectly fine — your answer can include decimal numbers!
