Module 7: Functions 🧠

🧠 From Variables to Functions 🧠

You already know variables. You have been using them since Module 1! A variable stores a piece of information. Simple!

Writing Instructions Once

Now meet a function. A function is a named container for instructions. You write the instructions once, give them a name, and use that name whenever you need them to run.

In Pseudocode It Looks Like This:

FUNCTION "show name"
  PRINT "My name is ", "child name"
END FUNCTION

To make it run, you CALL it:

CALL "show name"

Adding Parameters

What if we want the function to work with DIFFERENT information each time? We use a parameter! A parameter is information you pass INTO the function when you call it.

FUNCTION "show name" WITH "child name"
  PRINT "My name is ", "child name"
END FUNCTION

CALL "show name" WITH "David"
CALL "show name" WITH "Samantha"

💡 One function — works for everyone! 🎉

🌳 The Function Tree 🌳

Variables can live in different places — and that changes what they can do!

Where Variables Live

  • OUTSIDE a function: Available everywhere.
  • INSIDE a function: Only lives while the function runs.
  • As a PARAMETER: Information given to the function each time it is called.

The Full Tree Working Together:

SET "school name" TO "Greenfield Primary"

FUNCTION "introduce child" WITH "child name", "desk number"
  PRINT "School: ", "school name"
  PRINT "My name is ", "child name"
  PRINT "I sit at desk ", "desk number"
END FUNCTION

CALL "introduce child" WITH "David", 5
CALL "introduce child" WITH "Samantha", 12

💡 Write once. Use anywhere. That is the power of functions! 🚀
