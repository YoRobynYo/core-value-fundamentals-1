## THE JOURNEY

Children visit the Popcorn Factory and learn how it works by helping with various tasks.
Each module represents a deeper level of understanding.

**The Progression:**
1. **Variables** → Learning the MAP (where is everything?)
2. **Operators** → Counting SUPPLIES (what quantities?)
3. **Conditions** → Understanding WORKFLOW (when, how, why?)
4. **Logical Operators** → COMBINING decisions (AND both / OR either)
5. **Loops** → REPETITIVE tasks (doing things over and over)
6. **Functions** → SPECIALISED jobs (tasks we do often) ← to be designed
7. **Arrays** → MANAGING LISTS (tracking collections) ← to be designed
8. **Objects** → ORGANISING INFORMATION (related data together) ← to be designed

## MODULE 1: VARIABLES 🧩
### Theme: Getting Organised — Where is Everything?

**Opening:**
"Welcome to our classroom! Before we can start learning, we need to get organised.
Just like we label our desks, cubbies, and supplies — in coding we use VARIABLES to
label and store information. Let's start by getting to know each other and mapping our classroom!"

**Exercise Flow:**
1. Classroom Introduction — who am I, where do I sit?
2. Supply Cupboard Mapping — where are the supplies?
3. Getting to Know Our Friends — storing information about classmates
4. Teacher Information — who teaches us and what?
5. Daily Schedule — organising our time

**Closing:**
"Great work! You've learned how to store and organise information using variables.
Next we'll use these skills to COUNT things and do CALCULATIONS. But first —
guess what? We're going on a field trip to the Popcorn Factory! 🍿"

## MODULE 2: OPERATORS ➕➖✖️➗
### Theme: Counting Supplies — What Quantities Do We Have?

**Opening:**
"Welcome back! Yesterday you learned WHERE everything is.
Today you're going to COUNT what's inside each area! The inventory manager
needs your help calculating supplies. Let's start counting! 🍿"

**Exercise Flow:**
1. Counting Corn Kernels — addition
2. Bags Used vs Remaining — subtraction
3. Cases for Shipping — division
4. Flavouring Batches — multiplication
5. Complete Inventory Report — all operators combined

**Closing:**
"Excellent counting! Now that you know WHERE things are and HOW MUCH we have,
next we'll learn about the PROCESS of making popcorn — that's when things get
really interesting! 🍿"

## MODULE 3: CONDITIONS ❓
### Theme: Understanding the Workflow — When, How, and Why?

**Opening:**
"Welcome to your third day at the Popcorn Factory! You know where everything is
and you can count supplies. Now it's time to learn HOW we actually MAKE popcorn!
Today you'll help with production from start to finish. Let's head to the Great Hall! 🍿"

**CONDITIONS PART 1 — Greater than / Less than (>, <)**
Theme: Arrival and Preparation Checks
Context: Before production begins — are there enough children? Is the workspace ready?

**CONDITIONS PART 2 — Equal to / Not equal to (=, ≠)**
Theme: Getting Organised — Exact Matching
Context: Assigning children to stations — is everyone at the CORRECT station?

**CONDITIONS PART 3 — Greater/Less than or equal to (>=, <=)**
Theme: Ingredients Room — Threshold Checks
Context: Do we have ENOUGH supplies? Are we OVER capacity?

**CONDITIONS PART 4 — If / Else**
Theme: Checklist Verification — Making Decisions
Context: For each item on the checklist — IF ready THEN check off ELSE get more

**CONDITIONS PART 5 — Else If**
Theme: Quality Control — Multiple Outcomes
Context: Popcorn quality has multiple grades — not just pass/fail

**Closing (end of all Conditions):**
"Congratulations! You've mastered all the decision-making skills needed in the factory!
You can compare values, check exact matches, verify thresholds, make backup plans,
and handle multiple outcomes. The production manager is SO impressed!
Next — LOOPS! Because in a factory we don't do things just once — we do them hundreds of times! 🍿"

---

## MODULE 4: LOGICAL OPERATORS 🔒
### Theme: Combining Decisions — AND Both / OR Either

**Opening:**
"Welcome back! You've mastered conditions. Now we're going to learn how to
COMBINE conditions together. Sometimes you need BOTH things to be true.
Sometimes EITHER one will do. Let's learn when to use each! 🍿"

**LOGICAL OPERATORS PART 1 — AND**
Theme: Safety Checks — BOTH conditions must pass
Context: The factory has strict safety rules — machines only run if ALL checks pass

**LOGICAL OPERATORS PART 2 — OR**
Theme: Flexible Options — EITHER condition is enough
Context: The factory can keep running if ANY one of several backup options works

**Closing:**
"Brilliant! You can now combine conditions like a pro. Next we tackle LOOPS —
the most powerful tool for handling repetitive factory tasks! 🍿"

---

## MODULE 5: LOOPS 🔁
### Theme: Repetitive Tasks — Doing Things Over and Over

**Opening:**
"Welcome to the busiest part of the factory! Everything here happens over and
over — counting bags, checking machines, tracking totals. Instead of writing
the same code hundreds of times — we use LOOPS! 🍿"

**LOOPS PART 1 — Basic Counting**
Theme: Counting supplies and tracking numbers with addition and multiplication

**LOOPS PART 2 — Subtraction and Division**
Theme: Tracking items decreasing over time and sharing equally

**LOOPS PART 3 — Loops with Conditions**
Theme: Smarter loops — skip items, stop early, only count certain things

**LOOPS PART 4 — Accumulators**
Theme: Using a variable OUTSIDE the loop to track running totals

**Closing:**
"Amazing work! You've completed the Loops module — one of the most powerful
tools in coding. You're ready for the next big challenge! 🍿"

## NARRATIVE PRINCIPLES

1. Always say "children" not "kids"
2. Always say "coding" not "programming"
3. Use encouraging, supportive language
4. Frame tasks as "helping" not "doing work"
5. Acknowledge progress frequently
6. Make connections to real factory operations
7. Use age-appropriate explanations (10-12 years old)
8. Add MORE story not less — the narrative is essential to learning
9. Children must feel like PARTICIPANTS not just command-followers
10. Never use robotic command language — always give context for WHY

- Rule 6 always applies — never introduce concepts not yet taught
