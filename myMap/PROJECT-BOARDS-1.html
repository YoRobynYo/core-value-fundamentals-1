Unified Project Board
📁 Pipeline Files
⚙️ Config
package.json
Node project config — lists dependencies and npm scripts
pipeline-config.json
Gold standard rules — every pipeline file reads this. Change once, everything updates
↳ feeds all build files
locked-modules.json
List of completed modules that the pipeline must never overwrite
↳ read by rebuild-all.js, rebuild-new.js
🔁 Build Files
agent-pipeline.js
Master orchestrator — runs the AI waterfall chain (OpenRouter → Gemini → Ollama) and triggers fallback when rate limits hit
↳ calls build-html.js, monitors limits
build-spec.js
Generates module spec files from templates using AI. Feeds into the HTML build step
↳ reads templates → outputs spec files
build-spec-ollama.js
Same as build-spec.js but routes to local Ollama model when cloud APIs are unavailable
build-html.js
Core build file — reads config + templates, sends prompt to AI, receives module HTML, writes to output folder
↳ reads config → AI → output/
rebuild-all.js
Loops through every module and rebuilds from scratch using current config. Supports --from flag to start from a specific module
rebuild-new.js
Only builds modules that don't exist yet — skips completed ones. Used when adding new modules without a full rebuild
rebuild-fix-errors.js
Targets only modules that failed the audit — reads audit output, rebuilds problem files, leaves clean files untouched
↳ reads audit-html.js output
rewrite-spec.js
Rewrites existing spec files when the template changes — cloud version
rewrite-spec-ollama.js
Rewrites existing spec files using local Ollama model
update-template.js
Pushes template changes across all existing built files without a full rebuild
fix-template.js
Targeted template repair — fixes specific structural issues in output files
🔍 Monitor & Audit
audit-html.js
Runs quality checks on all output HTML — flags banned words, missing elements, structural errors
↳ feeds rebuild-fix-errors.js
pipeline-monitor.js
Watches the pipeline in real time — shows which AI is active, when limits hit, when waterfall triggers to next AI
📜 Course Scripts
scripts/main.js
Main course interaction script — powers exercise UI logic across all modules
scripts/chat.js
Handles the in-course chat/hint system
scripts/scroll.js
Manages scroll behaviour within module pages
🎨 Styles
styles/scss/
Source SCSS files — edit these, not the compiled CSS
styles/css/
Compiled CSS output — auto-generated, do not edit directly
📚 Reference Docs

roadmap.md

Research Findings.pdf

📌 Master Templates
Core Templates
1-HOW-TO-REWRITE-MODULE-SPECS.md
Step-by-step instructions for rewriting or updating module spec files — used when templates change
2-MODULE-0-EXERCISE-TEMPLATE-MASTER.md
The master exercise template — defines the structure every exercise must follow including layout, code blocks, stepped instructions

Spec template — the format every module spec file must follow before being passed to the HTML builder

The reference quality benchmark — a perfect finished exercise the AI must match in structure, tone, and layout

AA-JS-HINTS-PATTERN-GUIDE.md
Pattern guide for the hints system — defines how hints are written, structured, and triggered across all modules
Templates-Breakdown.md
Master reference explaining what each template does, what order they're used in, and how they connect to each other
⚡ Build Actions
01 · OPEN BRIEFING
Read PROJECT-BRIEFING.md and load full context into the AI session before any build work
02 · BUILD SPEC
Generate the module spec file for the target module
node build-spec.js
03 · RUN PIPELINE
Generate the exercise HTML — pipeline handles AI waterfall automatically
node agent-pipeline.js
04 · AUDIT & REVIEW
Run quality checks on output, fix any flagged errors
node audit-html.js
05 · FIX ERRORS
Rebuild only the modules that failed audit — leaves clean files untouched
node rebuild-fix-errors.js
